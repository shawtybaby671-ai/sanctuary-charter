# RFC Index

This project uses an Issues-based RFC process.

## Numbering
RFCs are numbered sequentially: `RFC-0001`, `RFC-0002`, etc.  
Include the RFC number in the issue title once assigned (e.g., `RFC-0001: Clarify Continuity definition`).

## Where RFCs live
RFCs are GitHub Issues labeled `rfc`.

## Decision process (v0.2)
- RFCs should be written to be reviewable and testable (include verification/measurement).
- Maintainers summarize discussion and record the decision + rationale in `docs/DECISIONS.md`.
- Approved RFCs result in a pull request updating the relevant documents.
